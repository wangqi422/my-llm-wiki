## 二、枪火特效制作流程

### 2.1 预渲染CG中的枪口焰制作流程

```
┌──────────────────────────────────────────────────────────┐
│                  枪口焰完整制作管线                        │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Phase 1: 参考收集 (Reference)                            │
│  ├─ 实拍高速摄影素材 (1000-10000fps慢动作)                │
│  ├─ ActionVFX等素材库高清参考                              │
│  └─ 不同枪型/口径/环境下的真实枪焰差异记录                 │
│           ↓                                              │
│  Phase 2: 概念设计 (Concept)                              │
│  ├─ 2D概念帧（确定形状、颜色、规模）                       │
│  ├─ 时序节奏设计（爆发/扩散/消散三阶段时长）               │
│  └─ 交互光照范围与强度设计                                │
│           ↓                                              │
│  Phase 3: 3D制作 (Production)                             │
│  │                                                       │
│  │  ┌─ 方案A: Houdini 流体模拟 ──────────────────┐       │
│  │  │  • Pyro Solver 模拟火焰/烟雾                │       │
│  │  │  • 发射源设置（枪口几何体作为Source）         │       │
│  │  │  • 速度场/温度场/密度场参数调节              │       │
│  │  │  • 模拟缓存 → VDB 体积数据                  │       │
│  │  └──────────────────────────────────────────────┘       │
│  │                                                       │
│  │  ┌─ 方案B: 粒子系统 (Maya/UE/Unity) ──────────┐       │
│  │  │  • 多层粒子发射器设置                        │       │
│  │  │  • Layer1: 核心闪光 (Billboard)              │       │
│  │  │  • Layer2: 火花飞溅 (GPU Particles)          │       │
│  │  │  • Layer3: 烟雾拖尾 (Sprite Sequence)        │       │
│  │  │  • 每层独立的生命周期/颜色/大小曲线          │       │
│  │  └──────────────────────────────────────────────┘       │
│  │                                                       │
│  │  ┌─ 方案C: Flipbook序列 ──────────────────────┐       │
│  │  │  • Houdini模拟 → 渲染为Sprite Sheet         │       │
│  │  │  • 8×8或16×16帧序列贴图                     │       │
│  │  │  • 实时播放，性能友好                        │       │
│  │  └──────────────────────────────────────────────┘       │
│           ↓                                              │
│  Phase 4: 光照交互 (Lighting)                             │
│  ├─ 创建伴随动态点光源（与枪焰同步创建/销毁）             │
│  ├─ 光源颜色随时间变化（白热→橙黄→暗红）                  │
│  ├─ 光源强度曲线（瞬间峰值→快速衰减）                     │
│  └─ 环境阴影投射设置                                      │
│           ↓                                              │
│  Phase 5: 渲染 (Rendering)                                │
│  ├─ CG预告片: Mantra/Arnold/V-Ray/Redshift 离线渲染       │
│  ├─ 实时引擎: UE5 Niagara / Unity VFX Graph               │
│  └─ 分层渲染: 火焰层/烟雾层/光照层分开输出                 │
│           ↓                                              │
│  Phase 6: 合成 (Compositing)                              │
│  ├─ Nuke/After Effects 中分层合成                          │
│  ├─ 添加Bloom/Glow/Lens Flare增强                         │
│  ├─ 与实拍/CG背景融合调色                                 │
│  └─ 最终输出                                              │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 2.2 各公司的枪火制作技术路线

| 公司 | 软件栈 | 枪火方案 | 特点 |
|------|--------|---------|------|
| **Blur Studio** | 3ds Max + XSI + 自研工具 | Houdini Pyro → 3ds Max 合成 | 电影级流体模拟，单帧渲染时间长 |

## 三、爆炸特效制作流程

### 3.1 CG 预告片中的爆炸制作全管线

```
┌──────────────────────────────────────────────────────────┐
│                  爆炸特效完整制作管线                      │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Step 1: 参考与设计                                       │
│  ├─ 真实爆炸高速摄影参考（军事纪录片/ActionVFX素材）       │
│  ├─ 确定爆炸类型（手雷/RPG/车辆/建筑/航弹）               │
│  ├─ 规模设计（半径/高度/持续时间）                         │
│  └─ 分层概念图（闪光/火球/冲击波/碎片/烟尘/余烬）         │
│           ↓                                              │
│  Step 2: 破坏模拟 (Destruction)                           │
│  ├─ 工具: Houdini RBD / SideFX RBD Bullet Solver          │
│  ├─ 原始模型 → Voronoi 碎片化（控制碎片大小分布）          │
│  ├─ 约束系统设置（Glue/Spring/Soft约束）                   │
│  ├─ 爆炸力场施加（爆心位置/力度/方向）                     │
│  ├─ 碎片物理模拟（重力/碰撞/摩擦/弹跳）                   │
│  └─ 缓存输出 → Alembic/VDB                               │
│           ↓                                              │
│  Step 3: 火焰模拟 (Pyro - Fireball)                       │
│  ├─ 工具: Houdini Pyro Solver / Axiom Solver               │
│  ├─ 爆炸源几何体 → 速度场发射                              │
│  ├─ 参数调节:                                              │
│  │   • Temperature (温度) → 控制火焰亮度和上升力            │
│  │   • Fuel (燃料) → 控制燃烧持续时间                      │
│  │   • Density (密度) → 控制烟雾浓度                       │
│  │   • Turbulence (湍流) → 控制翻滚细节                    │
│  │   • Dissipation (消散) → 控制消失速度                   │
│  ├─ 分辨率: 通常 0.02-0.05 体素大小                        │
│  └─ 缓存输出 → VDB序列                                    │
│           ↓                                              │
│  Step 4: 烟雾模拟 (Pyro - Smoke Plume)                    │
│  ├─ 独立于火球的二次模拟（更大尺度/更长时间）               │
│  ├─ 热浮力驱动上升                                        │
│  ├─ 风场影响（全局+局部湍流）                              │
│  └─ 缓存输出 → VDB序列                                    │
│           ↓                                              │
│  Step 5: 碎片粒子 (Debris/Shrapnel)                       │
│  ├─ 从RBD模拟中提取碎片运动数据                            │
│  ├─ 补充小型碎屑粒子（石子/灰尘/金属片）                   │
│  ├─ 碎片尾迹烟雾（每个碎片拖尾小型Pyro）                   │
│  └─ 碎片落地时的二级尘土喷射                               │
│           ↓                                              │
│  Step 6: 冲击波 (Shockwave)                               │
│  ├─ 球形扩散的扭曲几何体                                  │
│  ├─ 或后期合成阶段用扭曲滤镜实现                           │
│  ├─ 冲击波掀起的环形地面灰尘                               │
│  └─ 冲击波对周围物体的推力（物理力场）                     │
│           ↓                                              │
│  Step 7: 渲染 (Rendering)                                 │
│  ├─ 体积渲染: Mantra / Karma / Arnold / Redshift           │
│  ├─ 分层输出:                                              │
│  │   • Fire Pass (火焰发光层)                              │
│  │   • Smoke Pass (烟雾遮挡层)                             │
│  │   • Debris Pass (碎片层)                                │
│  │   • Dust Pass (灰尘层)                                  │
│  │   • Light Pass (爆炸光照对环境的影响)                    │
│  │   • Shadow Pass (爆炸阴影)                              │
│  ├─ AOV输出: Depth/Normal/Velocity/Cryptomatte              │
│  └─ 高动态范围(HDR)渲染                                   │
│           ↓                                              │
│  Step 8: 合成 (Compositing)                               │
│  ├─ 工具: Nuke (电影级) / After Effects (快速迭代)          │
│  ├─ 分层合成火焰+烟雾+碎片+灰尘                           │
│  ├─ 添加: Bloom / Lens Flare / Heat Distortion              │
│  ├─ 调色: 爆炸暖色 vs 环境冷色对比                          │
│  ├─ 摄像机震动模拟                                        │
│  └─ 最终输出                                              │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

## 四、子弹交互（命中反馈）制作流程

### 4.1 完整制作管线

```
┌──────────────────────────────────────────────────────────┐
│              子弹命中交互 完整制作管线                     │

│                                                          │
│  Step 1: 材质定义系统                                     │
│  ├─ 为场景每个物体定义"物理材质标签"                       │
│  │   (混凝土/金属/木材/玻璃/泥土/雪/水/人体...)           │
│  └─ 材质标签决定命中时触发哪套特效                         │
│           ↓                                              │

│  │  └──────────────────────────────────────────────┘       │
│  │                                                       │
│  │  ┌─ 金属命中 ──────────────────────────────────┐       │
│  │  │  Emitter 1: 高速火花 (拉伸粒子, 橙白色)      │       │

│  │  ┌─ 木材命中 ──────────────────────────────────┐       │
│  │  │  Emitter 1: 木屑爆射 (锥形, 浅黄色)          │       │
│  │  │  Emitter 2: 木质纤维 (细长条粒子)             │       │
│  │  │  Decal: 弹孔 (圆形贯穿/椭圆斜穿)             │       │
│  │  │  大口径: 触发预断裂模型                        │       │
│  │  │  Audio: 木质碎裂声                             │       │
│  │  └──────────────────────────────────────────────┘       │
│  │                                                       │
│  │  (玻璃/泥土/雪/水/人体... 各自分支类似)                │
│           ↓                                              │
│  Step 4: Decal 贴花系统                                   │
│  ├─ 根据命中角度投射贴花到表面                             │

│           ↓                                              │

│                                                          │

```

### 4.2 CG预渲染 vs 实时引擎的差异

| 维度 | CG预渲染 (预告片/过场) | 实时引擎 (游戏内) |
|------|----------------------|------------------|
| **模拟精度** | Houdini 高精度流体模拟 | GPU粒子+Flipbook序列 |
| **碎片数量** | 数千-数万碎片 | 数十-数百碎片 |
| **烟尘** | 体积渲染VDB | Billboard粒子近似 |
| **光照交互** | 全局光照+光追 | 简化动态点光源 |
| **Decal** | 高精度置换贴图 | 投射Decal贴花 |
| **渲染时间** | 每帧数分钟-数小时 | 每帧 <16ms (60fps) |
| **适用工具** | Houdini+Maya+Nuke | UE5 Niagara / Unity VFX Graph |
